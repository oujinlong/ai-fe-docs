3.eslintrc.js

Eslint 编码规范的配置文件 为了在开发过程成保持良好的编码习惯和统一的代码规范,我们会用到Eslint插件,但是在使用过程中我们需要添加或者删除一些规范就需要我们自行配置,那么eslintrc.js即为Eslint的配置文件

首先我们了解一下 Eslint 默认的规则:

  • 代码末尾不能加分号
  • 代码中不能存在多行空行
  • tab键不能使用，必须换成两个空格
  • 代码中不能存在声明了但未使用的变量

eslintrc.js 中 rules 用来配置规则

rules: {
    "规则名": [规则值, 规则配置]
}
